## 📌 一句话总结
Docker 是一个开源的应用容器引擎，让开发者可以打包应用及其依赖包到一个可移植的容器中，实现"一次构建，到处运行"。

## 🎯 它是做什么的？

Docker 就像是一个"标准化的搬家箱"：

- **环境一致性**：开发环境、测试环境、生产环境完全一致
- **快速部署**：秒级启动应用，无需繁琐的配置
- **资源隔离**：每个容器都有独立的文件系统和网络
- **版本管理**：像管理代码一样管理应用环境

## 🔑 核心概念

**镜像（Image）**
- 专业说：只读模板，包含运行应用所需的一切
- 通俗说：就像一个"模具"，可以用来生产无数个相同的"产品"

**容器（Container）**
- 专业说：镜像运行时的实例
- 通俗说：用模具生产出来的具体产品，每个产品都是独立的

**仓库（Registry）**
- 专业说：存储和分发镜像的服务
- 通俗说：就像 GitHub，只不过存的是镜像而不是代码

## 💡 实际应用场景

**场景 1：解决环境配置问题**
- **痛点**：开发环境能跑，生产环境报错
- **方案**：将应用及其依赖打包成 Docker 镜像
- **效果**：在任何支持 Docker 的机器上都能正常运行

**场景 2：微服务架构**
- **痛点**：多个服务相互干扰，难以管理
- **方案**：每个服务运行在独立的容器中
- **效果**：资源隔离，独立部署，易于扩展

**场景 3：CI/CD 自动化**
- **痛点**：构建、测试、部署流程复杂且易出错
- **方案**：使用 Docker 容器化整个构建流程
- **效果**：标准化流程，提高部署效率

## 🚀 快速上手

### 安装 Docker
```bash
# Ubuntu/Debian
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh

# 启动 Docker 服务
sudo systemctl start docker
sudo systemctl enable docker
```

### 第一个 Docker 容器
```bash
# 运行 Hello World 容器
docker run hello-world

# 运行 Nginx 服务器
docker run -d -p 80:80 --name my-nginx nginx

# 查看运行中的容器
docker ps
```

### 构建自定义镜像
```dockerfile
# Dockerfile
FROM node:14
WORKDIR /app
COPY package*.json ./
RUN npm install
COPY . .
EXPOSE 3000
CMD ["npm", "start"]
```

```bash
# 构建镜像
docker build -t myapp:1.0 .

# 运行容器
docker run -d -p 3000:3000 --name myapp myapp:1.0
```

## 📚 常用命令

```bash
# 镜像操作
docker images                    # 列出所有镜像
docker pull nginx:latest        # 拉取镜像
docker rmi <image_id>           # 删除镜像

# 容器操作
docker ps                       # 列出运行中的容器
docker ps -a                    # 列出所有容器
docker stop <container_id>      # 停止容器
docker rm <container_id>        # 删除容器
docker logs <container_id>      # 查看日志

# 清理操作
docker system prune -a          # 清理所有未使用的资源
```

## ⚠️ 注意事项

- **学习曲线**：⭐⭐⭐☆☆（需要理解 Linux 和网络基础知识）
- **适用场景**：微服务、云原生应用、CI/CD 流水线
- **不适用场景**：高性能计算、需要直接访问硬件的应用
- **安全建议**：
  - 使用官方镜像或可信镜像
  - 定期更新镜像版本
  - 避免在容器中运行 root 用户
  - 限制容器资源使用
